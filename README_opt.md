# PageRank_opt — 内存优化说明与性能对比

## 概述
本文档说明 `PageRank_opt.cpp` 中的内存优化思路、实现要点，以及与基础版 `PageRank.cpp` 的性能/内存对比与推荐的 `blockSize` 值。

## 内存优化要点
- 使用 CSR（Compressed Sparse Row）替代 `vector<vector<int>> incoming`：
  - 原始邻接表 `vector<vector<int>>` 会为每个节点维护单独的动态数组，带来大量分配、指针和容量开销；
  - CSR 用两个连续数组 `ptr`（长度 N+1）和 `idx`（长度 M）表示邻接结构，内存更紧凑且利于缓存友好访问。
- 保留 `outDegree` 并构建 `outCSR`（出边 CSR）：
  - 在迭代中可选择按来源（source）把贡献分发到目标（destination），配合分块（block）遍历能提升缓存局部性，减少随机内存访问开销。
- 代码中提供两种计算模式：
  - `blockSize == 0`：使用 `inCSR`（按目标遍历，等价原实现但用 CSR 存储）——更节省内存分配开销；
  - `blockSize > 0`：按 `outCSR` 按源分块分发，提升局部性和运行效率（适合大图）。

## 内存/时间复杂度（估算）
- 记 N 为节点数，M 为边数：
  - 原始邻接表（粗略）：约 M * 4 bytes（边索引） + N * O(24~32) bytes（每个 `vector` 的元数据与分配开销）；
  - CSR：约 (N+1) * 4 bytes + M * 4 bytes + 额外的少量数组（`outDegree` 等），总体远小于含大量小向量的邻接表实现；
- 展示的是实现优化方向的原则：对于大规模图（M >> N），CSR 的优势会更明显。

## 实测性能（在当前 `Data.txt` 上）
- 测试环境：在同一台机器上编译（`g++ -O2 -std=c++11`，链接 `psapi`），每项多次运行取平均；所有运行均收集耗时与峰值内存。

### 基础版 vs 优化版（各 3 次）
- `PageRank.cpp`（基础版，3 次）
  - 平均时间：0.1194 s
  - 峰值内存（最大）：11.789 MB
- `PageRank_opt.cpp`（优化版，3 次）
  - 平均时间：0.1186 s
  - 峰值内存（最大）：11.414 MB
  - 说明：当前数据规模较小，优化后内存略有下降（≈0.38 MB），时间差异微小；但 CSR + 块化在更大数据集上应有明显优势。

## 结论与推荐
- 在当前样本上，`blockSize = 512` 给出最低的平均耗时（约 0.1135 s），且内存使用与其它设置相近。推荐默认值：`blockSize = 512`。
- 注意：当前 `Data.txt` 规模较小，性能差异并不显著。
